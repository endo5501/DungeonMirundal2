# DungeonMirundal

> **Note:** これはプロジェクト初期(2026-04 頃)に作成された設計スナップショットです。最新仕様は `openspec/specs/` を参照してください。

## プロジェクト概要

* Godot製の(Wizardlyライクな)ダンジョン探索RPGを作成する
* ユーザの操作を簡略化するため、次のルールを追加
    * 地上へ戻ったらHPを全回復, 死亡以外の状態異常も解除
* ダンジョンは固定ではなく自動生成
* ゲームで使用する各種データ(職業・種族などのステータス、モンスター、アイテムなど)はゲーム内に埋め込むのではなく、外部ファイルとして管理してカスタマイズ出来るようにする

## UIメニューの構造

### スタート画面

* 新規ゲーム
* 前回から(最後にセーブしたデータから)
* ロード(セーブデータを選択)
* ゲーム終了

### 地上画面

地上マップは以下の要素から構成されている。UIの見た目としては画面左側に各施設メニュー選択ボタンを表示。右側にイラストを表示する(可能であれば、施設選択ボタンで選択中の施設のイラスト)
各地上施設を選択して決定すると、各設備の選択メニューが表示される。

* **冒険者ギルド**
    * キャラクター作成画面
    * パーティ編成画面
    * キャラクター一覧
    * パーティ名を変更

* **商店**
    * アイテム購入
    * アイテム鑑定
    * アイテム売却

* **教会**
    * 蘇生

* **ダンジョン入口**
    * ダンジョン選択
    * ダンジョン新規生成
    * ダンジョン破棄

### 設定画面

[ESC]キーを押下すると、設定画面が表示される。この設定画面への遷移は地上、ダンジョンでもどちらでも可能とする

* パーティメニュー
    * メンバーのステータス表示
    * メンバーのアイテム操作管理
    * メンバーの装備管理
* ゲームを保存
* ゲームをロード
* 設定
* 終了

### ダンジョン画面

* Wizardly風ダンジョン
* ダンジョンは自動生成で作成する。 アルゴリズムは docs/reference/dungeon_generator.py を参考に作成する
* ダンジョン内で可能な操作は移動のみとする(ひとまず)
* レイアウト
    上部:ダンジョンの3D表示
    下部:現在のパーティ編成表示(前列:3名,後列:3名)
* オートマッピングあり、上からの探索済み領域を表示するスモールマップを画面右上に表示
