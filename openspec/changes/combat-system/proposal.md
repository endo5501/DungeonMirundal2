# Change: 戦闘システム

## Summary

monster-and-encounter で発生したエンカウントに対して、Wizardry 風のターン制戦闘を実装する。戦闘画面はダンジョン画面のオーバーレイとして表示し、シーン遷移は行わない。アイテム・装備はダミーデータ（ハードコード数点）を使用し、本番データへの差し替えと戦闘バランス最終調整は後続の change で行う。

## Scope

- 戦闘ロジック（ターン制）
  - TurnEngine（戦闘状態の遷移管理）
  - コマンドリゾルバ（攻撃、防御、逃走）
  - 行動順決定（素早さ基準）
  - ダメージ計算（ダミー装備の攻撃力/防御力を反映）
  - 勝敗判定
  - 経験値獲得・レベルアップ
  - キャラクター死亡処理
- 戦闘UI（Wizardry 風固定レイアウト、ダンジョン画面オーバーレイ）
  - モンスター表示パネル
  - コマンド選択メニュー
  - 戦闘ログ表示
  - 結果画面（経験値、ドロップ枠）
  - 入力フォーカス制御（ESCメニュー等との競合解消）
- ダミーアイテム
  - 武器・防具を数点ハードコードし、戦闘バランスを仮調整
  - items-and-economy で本番データに差し替え可能な最小インターフェース
- 既存スタブの差し替え
  - monster-and-encounter のスタブ・オーバーレイを本 change の戦闘UIに置き換え

## Non-goals

- 本番アイテムデータ・商店・教会（items-and-economy）
- 魔法・スキルシステム（必要であれば別 change で）
- 戦闘バランスの最終調整（全 change 完成後に実施）
- アニメーションや演出強化（本 change は固定レイアウトのみ）

## Dependencies

- monster-and-encounter: モンスターデータとエンカウント発生
- character-and-party-system (archived): 戦闘参加キャラクターのステータス

## Risks

- Wizardry 風 UI のレイアウト設計（情報量と視認性のバランス）
- オーバーレイ時の入力制御（ESCメニュー・ダンジョン操作との排他制御）
- ターン制状態管理の複雑さ（複数キャラクター、死亡・戦闘不能の扱い）
- ダミーアイテム → 本番アイテムの差し替え時の影響範囲を最小化する設計
