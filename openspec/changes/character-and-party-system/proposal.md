# Change: キャラクター＆パーティシステム

## Summary

キャラクターの作成・管理およびパーティ編成の仕組みを構築する。種族・職業などのデータは外部ファイルで定義し、ここで外部データファイルの形式とローダーの設計を確定する。

## Scope

- 外部データファイル形式の決定とローダー実装
- 種族・職業データの定義と読み込み
  - 種族（ステータス補正、特性など）
  - 職業（成長パラメータ、装備可能カテゴリなど）
- キャラクター作成ロジック
  - 名前・種族・職業の選択
  - 初期ステータスの決定
- パーティ編成ロジック
  - 前列3名・後列3名の編成
  - メンバーの入替え
- 冒険者ギルドUI
  - キャラクター作成画面
  - パーティ編成画面
  - キャラクター一覧
  - パーティ名変更

## Non-goals

- レベルアップ・経験値（戦闘システムと連動するため combat-system 以降）
- アイテム・装備の実装（items-and-economy）

## Dependencies

- project-foundation-and-dungeon-generation: プロジェクト基盤（Godotプロジェクト、GUT）

## Risks

- 外部データファイル形式の選定（JSON / YAML / Godot Resource 等）
- データバリデーションの設計
