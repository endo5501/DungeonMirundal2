# Change: 戦闘システム

## Summary

ダンジョン内でのモンスターとのエンカウントおよびターン制戦闘を実装する。アイテム・装備は本changeではダミーデータを使用し、戦闘バランスの最終調整は全体完成後に行う。

## Scope

- モンスターデータ定義
  - 外部データファイルでのモンスター定義
  - ステータス（HP、攻撃力、防御力、素早さ等）
  - 出現条件（ダンジョン階層、エリア等）
- エンカウントロジック
  - ランダムエンカウント（移動時の発生判定）
  - モンスターパーティの編成ロジック
- 戦闘ロジック（ターン制）
  - コマンド選択（攻撃、防御、逃走等）
  - 行動順決定（素早さ基準）
  - ダメージ計算
  - 勝敗判定
  - 経験値・レベルアップ
  - キャラクター死亡処理
- 戦闘UI
  - モンスター表示
  - コマンド選択メニュー
  - 戦闘ログ表示
  - 結果画面（経験値、ドロップ等）
- ダミーアイテム（武器・防具）による仮バランス

## Non-goals

- 本番アイテムデータ（Change 8）
- 魔法・スキルシステム（必要であれば別changeで）
- 戦闘バランスの最終調整

## Dependencies

- Change 5: 地上画面 + 画面遷移（ダンジョンからの遷移管理）
- Change 4: キャラクター＆パーティシステム（戦闘参加キャラクターのデータ）

## Risks

- 戦闘バランスの仮調整（ダミーアイテムでの適切な難易度設定）
- 戦闘UIのレイアウト設計
- ターン制戦闘の状態管理の複雑さ
